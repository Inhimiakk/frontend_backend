Сокеты

TCP/UDP, клиент-сервер, порты.

Учебный материал

Сокет — точка связи между двумя программами по сети. Обычно есть серверный сокет (слушает порт) и клиентский (подключается).

  • TCP — надёжная доставка, порядок сообщений сохраняется (веб, почта).
  • UDP — быстрее, но без гарантии доставки (стримы, игры).
  • Порт — номер сервиса на машине (например 80 для HTTP).
  • Схема: сервер bind → listen → accept, клиент connect → send/recv.
